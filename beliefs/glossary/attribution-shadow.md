# attribution shadow

The enforceable remainder of a runtime act that the repository itself never witnesses. Model selection is the motivating case: which model performs a motion is decided at run time and leaves no committed artifact a verifier could gate, so a rule about *selection* cannot be machine-enforced — but the [provenance](/beliefs/glossary/provenance.md) field naming the producing model **is** committed and checkable. Requiring that attribution is the shadow the unenforceable act casts into the repo: you cannot audit whether capability was matched to a motion unless the artifact says which model performed it. Introduced by the [capability-matched model selection](/beliefs/glossary/capability-matched-model-selection.md) doctrine, which proposes a thin attribution policy as its path to auditability.
