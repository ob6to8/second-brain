# capability-matched model selection

Agents allocate model capability by the motion's epistemic weight rather than by habit or availability. It is the model-layer application of [orchestration](/beliefs/glossary/orchestration.md) — strong-model attention is spent the way the operator's own attention is. Deliberately a [doctrine](/beliefs/glossary/doctrine.md), not a policy: selection is a runtime act enforceable only through its [attribution shadow](/beliefs/glossary/attribution-shadow.md). Canonically defined in the [capability-matched model selection doctrine](/meta/doctrine/capability-matched-model-selection.md).
