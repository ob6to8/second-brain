# document (OKF)

A single markdown file with YAML frontmatter that is the unit of knowledge in
this bundle; its id is its path minus `.md`. Canonically defined by the
[document-anatomy policy](/meta/policy/document-anatomy.md).

The OKF spec calls this unit a *concept document*. This bundle deliberately says
**document**: the anatomy is purely structural, and bare *concept* is reserved
for the controlled `type: concept` (a definition or mental model). This entry
was formerly titled *concept (OKF)*; its stable id is unchanged.

*See also:* [concept (type)](/beliefs/glossary/concept-type.md) — the controlled `type` whose name this unit sense is deliberately kept distinct from.
