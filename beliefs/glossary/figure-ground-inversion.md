# figure-ground inversion

Applied to system design, the term names two architectures built from the same
parts with opposite emphasis: what one system treats as the product, the other
treats as an accessory. In this brain: the relation between
[eve](/beliefs/glossary/vercel-eve.md)-style agent frameworks and this bundle —
both are a directory of markdown under git, but eve is an *agent* carrying
knowledge as an accessory while this bundle is *knowledge* hiring agents as
transient labor; the [vercel-eve-comparison analysis](/meta/analysis/vercel-eve-comparison.md)
traces how durability, typing, duplication, and human-in-the-loop each invert
downstream of that one swap.
