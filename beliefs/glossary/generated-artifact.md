# generated artifact

An output file produced mechanically by compiling some source of truth, rather
than written by hand — the ordinary build-system sense (compiled binaries,
bundled assets, generated code).

**In this brain:** a specific, load-bearing pattern — a file **committed to the
repo** yet **compiled from elsewhere** and **guarded against drift**. Each such
artifact pairs a generator with a `--check` mode: the generator (`mix brain.<x>`)
renders the file from its source, and `mix brain.<x> --check` re-renders in memory
and fails the [gate suite](/beliefs/glossary/gate-suite.md) if the checked-in copy
differs. This converts "remember to regenerate" from a discipline into a
structural guarantee, so **the fix for a stale artifact is never to edit it** — it
is to rerun the generator. Members include the
[compiled contract](/beliefs/glossary/compiled-contract.md) (`CLAUDE.md` from the
policies), the registry (`meta/registry.md` from the ids), the
[code map](/beliefs/glossary/code-map.md) (`meta/code-map.md` from `lib/`
docstrings), and the derived dev-history (whose check is
[lag-tolerant](/beliefs/glossary/lag-tolerant-check.md)). The placement rule for
what may be a generated artifact is [write-time vs build-time](/beliefs/glossary/write-time-vs-build-time.md):
compilers derive, checkers compare, and only authored judgment is written by hand.
