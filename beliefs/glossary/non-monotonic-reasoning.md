# non-monotonic reasoning

Reasoning in which learning more can force you to conclude *less*: a
conclusion held by default ("believe A unless there is evidence to the
contrary") is withdrawn when contrary evidence arrives, so the set of
conclusions does not grow monotonically with the set of premises — unlike
classical logic, where adding premises never removes consequences. In TMS
terms this is the *outlist*: a belief supported partly by the absence of other
beliefs. The mechanism behind default rules, retractable
[assumptions](/beliefs/glossary/assumption.md), and
[defeasible](/beliefs/glossary/defeasibility.md) argument generally.
