# orchestration

The engineering discipline of building software by **directing AI agents that
write the code**, rather than writing it oneself. Per Anthropic's 2026
role-transformation framing, its value domains are system architecture design,
agent coordination, quality evaluation, and strategic problem decomposition: the
human decomposes the problem, coordinates the agents, evaluates their output, and
provides strategic direction — "ensuring the system solves the right problems for
the right stakeholders" — which lets one person shepherd many features
simultaneously across a broader scope than hand-implementation allowed. In this
brain it is the direction named by the [engineer-as-orchestrator doctrine](/meta/doctrine/engineer-as-orchestrator.md);
the brain's own operating loop (agents file and organize; the operator ratifies
shape and evaluates gated output) is a practiced instance of it.
