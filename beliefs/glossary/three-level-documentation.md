# three-level documentation

An opt-in presentation scheme in which a document carries up to three tiers: a **plain** tier (a plain-spoken backstop for a reader who fails to intuit the technical register — see [constant tutorial mode](/beliefs/glossary/constant-tutorial-mode.md)), the **technical** tier (the canonical terse body, exactly as the brain's documents are written), and a [grounding tier](/beliefs/glossary/grounding-tier.md) (one level of abstraction below the source). The design rule is *one canonical level plus two anchored derivations*: the technical body is the single source of truth, the plain tier is re-rendered from it, and the grounding tier points below it — never three independently drifting texts. Canonically defined in the [three-level documentation plan](/meta/plans/three-level-documentation.md) (`status: accepted`; execution deferred).
