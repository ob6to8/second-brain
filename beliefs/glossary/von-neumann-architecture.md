# von Neumann architecture

The classic stored-program computer organization: a processing unit (ALU), a control unit driving the fetch-execute cycle, memory holding both program and data, and I/O. The standard four-component agent decomposition maps onto it almost exactly — model as ALU, loop as control unit (the fetch-execute cycle at macro scale), context plus external store as memory, tools as I/O — with the prompt playing the stored program's role, which is the term the decomposition omits.
