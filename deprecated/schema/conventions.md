# Conventions

## Filenames
- kebab-case, derived from title (sources) or claim (assertions)
- Example: `persistent-memory-for-openclaw-mem0-plugin.md`

## IDs
- Same as filename without extension
- Example: `persistent-memory-for-openclaw-mem0-plugin`

## Tags
- Lowercase, hyphenated
- Example: `context-engineering`, `multi-agent`

## Wikilinks
- Format: `[[id]]` — no path prefix, no extension
- Example: `[[persistent-memory-for-openclaw-mem0-plugin]]`

## Dates
- ISO 8601: `YYYY-MM-DD`

## Summaries and Claims
- Single sentence or short phrase, written for scanning
- Claims (assertions): one declarative sentence stating the belief
- Summaries (sources): 2-3 sentences max describing what the source contains

## Directories
- `sources/` — flat, no subdirectories. Tags handle taxonomy.
- `assertions/` — flat, no subdirectories.
- `schema/` — system rules (this directory)
- `templates/` — frontmatter templates
- `scripts/` — shell tooling
- `plans/` — architecture plans (historical)
- `intake/` — staging area
