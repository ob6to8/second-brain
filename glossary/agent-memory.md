# agent memory

The mechanisms by which an AI agent persists knowledge beyond its live context window, conventionally split into short-term (the context window itself), long-term (facts and preferences surviving across sessions), procedural (codified workflows), and working memory (in-task scratchpad). Implementations range from plain markdown files under version control to managed [vector databases](/glossary/vector-database.md); this brain is itself a file-based instance of the pattern.
