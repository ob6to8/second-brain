# Code context

Tools and techniques that build and serve structured **codebase context** —
knowledge graphs, indexes, call/dependency maps — to coding agents, so the agent
gets architectural understanding without exploring a repo file by file.

## References

- [GitNexus — client-side code knowledge graph for agent context](/knowledge/SWE/agentic-coding/code-context/gitnexus.md) — parses a codebase into a Tree-sitter-based knowledge graph (calls, imports, clusters, execution flows) and serves it to agents over MCP. `sb:b89ea1` _(reference)_
