# technical-communication

How to present technical ideas: exposition methods, tone, and the structure of
explanations — distinct from [argumentation](/knowledge/knowledge-management/argumentation/index.md)
(the structure of *justification*); this directory is about the structure of
*teaching and presenting*.

## Contents

- [Escalating-example exposition](/knowledge/knowledge-management/technical-communication/escalating-example-exposition.md) — a presentation playbook distilled from Yos Riady's ECS article: open inside one concrete scenario, escalate the same memorable example until the design visibly breaks, name the problem only after it is felt, then rebuild with the alternative using the same cast.
