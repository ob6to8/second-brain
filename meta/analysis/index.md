# Analysis

Point-in-time evaluations and decision-support write-ups — a question investigated
against evidence (often the live bundle itself), yielding findings and a
recommendation, filed so the reasoning and its conclusion persist. A separate
namespace from `plans` (intended *work* to execute) and `issues` (problems to
track): an analysis is a *reasoned judgment on a question*.

Each analysis is a `type: analysis` doc.

- [Agents as GenServers with a per-agent OKF mind: the two-tier memory model](/meta/analysis/agents-as-genservers-with-per-agent-okf-mind.md) — evaluates modeling swarm agents as supervised GenServers that each reference their own elixir-mind for persistence; finds the process model is nearly solved (it is essentially Jido) and the load-bearing choice is what "its own mind" means — a private per-agent **silo destroys the corpus's value** (no cross-agent learning, massive duplication), whereas a **two-tier** model (private working mind + one brokered shared canonical mind) is the only version that scales; process state is ephemeral so the on-disk OKF bundle is the durable crash-recovery substrate, and git-is-not-a-concurrent-DB forces the librarian broker's single serialized write lane (its correctness boundary); provenance-by-construction is the swarm's trust substrate and the precondition for the [escape-rate](/meta/plans/auto-intake-escape-rate-sampling.md) metric. - [The tier-3/4 interface and how an architect determines the loop is trusted](/meta/analysis/tier-3-4-interface-and-trust-determination.md) — against Cherny's adoption ladder, finds the tier-3/4 human interface is not a better chat window but an inverted one (the system initiates; you read by exception) — a telemetry dashboard fused to an exception inbox, with the machine plane carrying all the bandwidth and no human UI — and that "has tier 3/4 been reached" is not a judgment but a measurement: a low, stable, *sampled* defect-escape rate on an independent verification oracle that stays flat as supervision is withdrawn (sampling rate never zero; drift is a demotion signal). Shows the two questions are one apparatus, and names this bundle's gap — nothing samples `/research` auto-intake to compute an escape rate.
- [Code as a natural-language compilation target](/meta/analysis/code-as-natural-language-compilation-target.md) — evaluates the idea that a codebase should be regeneratable from its specs, tests, and invariants (code as a disposable compilation target of natural language, as Elixir compiles to binary); finds the direction right and already dogfooded (`CLAUDE.md` is compiled) but the "compilation" framing an overclaim — NL→code is a one-to-many lossy relation, not a meaning-preserving function, so the target is an *equivalence class* of programs; carries two intuition frames (tests are time-domain samples and invariants the Nyquist band-limit, with aliasing as a false-but-coherent mental model; and snapshot-vs-trajectory, where whole-codebase regeneration during a fix swaps class members and discards the emergent load-bearing set), and derives the [regenerate-the-change doctrine](/meta/doctrine/regenerate-the-change-not-the-system.md).
