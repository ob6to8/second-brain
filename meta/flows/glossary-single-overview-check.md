The connective doc for the glossary's **single-overview convention**: every term
file under [`/beliefs/glossary/`](/beliefs/glossary/index.md) carries exactly one
canonical overview — its `description` frontmatter — which the site renders as
the entry page's lede paragraph *and* the index's Terms gloss shows **verbatim**;
the body below the lede is **expansion-only**, never a restatement. The operator
asked for the convention to be *enforceable, not prose guidance*, and this flow
is the enforcement: `mix brain.glossary` splits it into a **deterministic half**
(the index is a generated artifact) and a **calibrated-heuristic half** (a
mechanical redundancy detector over body sentences), each gating exactly as hard
as its oracle allows.

## 1. The problem

A term's overview existed in **three hand-maintained variants** that drifted
independently: the `description` frontmatter (rendered as the entry lede), the
body's first paragraph (typically a lightly reworded second definition stacked
directly under the lede — the operator's actor-model screenshot showed the two
near-identical paragraphs on one page), and a third hand-written paraphrase as
the gloss in [`index.md`](/beliefs/glossary/index.md)'s Terms list. Redundancy
here is [DRY](/beliefs/glossary/dry.md) failure applied to definitions: copies
drift until one is wrong, and the reader can't tell which. The direction: one
overview, shown verbatim everywhere it appears, and bodies that only ever add.

## 2. The pipeline — how the check engages, progressively

```
 authoring time (per /add-to-glossary run, or any edit to a term file)
 ┌─────────────────────────────────────────────────────────────────────────┐
 │  agent writes/updates the term file                                     │
 │    description  = THE canonical overview (lede + index gloss)          │
 │    body         = expansion only (mechanism, examples, distinctions)   │
 │        │                                                                │
 │        ▼                                                                │
 │  mix brain.glossary --materialize                                       │
 │    regenerates index.md's ## Terms from the term files                  │
 │    (title-sorted case-insensitively, gloss = description verbatim)      │
 └────────┬────────────────────────────────────────────────────────────────┘
          ▼
 verification (same command, no flag — three checks in order)
 ┌─────────────────────────────────────────────────────────────────────────┐
 │  1 descriptions   every term file has a non-empty description   → FAIL │
 │  2 index sync     ## Terms == its re-derivation, byte-for-byte  → FAIL │
 │  3 repetition     per body sentence, content-word containment    │      │
 │                     ≥ 72%  near-restatement                     → FAIL │
 │                     55–72% heavy overlap (editorial band)       → warn │
 └────────┬────────────────────────────────────────────────────────────────┘
          ▼
 enforcement surfaces (where a failure actually stops something)
 ┌─────────────────────────────────────────────────────────────────────────┐
 │  .githooks/pre-commit   → blocks the commit locally                     │
 │  ci.yml                 → fails the PR's checks                         │
 │  pages.yml (deploy gate)→ a drifted glossary is never rendered live     │
 └─────────────────────────────────────────────────────────────────────────┘
```

Warnings never fail the task — only `FAIL` results exit non-zero, the same
warn-vs-gate call the [route-tagging](/meta/policy/route-tagging.md) coverage
check makes.

## 3. The two halves

**The verbatim half — deterministic.** The index Terms section stopped being
maintained and became **materialized**, the same generated-artifact discipline as
[`CLAUDE.md`](/meta/flows/render-contract.md), [`meta/registry.md`](/meta/registry.md),
and the route-tag excerpt logs. `materialize/1` derives the section — one bullet
per term, `- [title](path) — description`, title-sorted case-insensitively —
preserving the hand-written prose above the heading; the check re-derives and
compares (a [self-consistency check](/beliefs/glossary/self-consistency-check.md),
tolerant only of trailing whitespace). Everything the old hand-kept list could
get wrong — a paraphrased gloss, a missing or extra bullet, ordering drift — is
one equality test, and the fix is always the same command.

**The no-repetition half — calibrated heuristic.** Semantic redundancy has no
exact oracle, so the check measures vocabulary instead of meaning, tuned so its
fail band is unambiguous. Per term file:

1. **Extract definition prose only:** the body up to any generated
   `## Thread excerpts — route-tagged log` section, minus the `#` heading and
   the *Seen in:* / *See also:* citation lines, with link URLs stripped (anchor
   text kept).
2. **Split into sentences** (on `.`/`!`/`?`/`;`, abbreviation-aware), skipping
   any with fewer than **8 content words** — too short to measure honestly.
3. **Normalize both sides:** lowercase, punctuation stripped, ~120 stopwords
   dropped, naive plural stemming (so "owns"/"own", "actors"/"actor" match).
4. **Score containment:** the fraction of the sentence's unique content words
   that also appear in the description. **≥ 72% fails** — that sentence is a
   near-restatement of the lede; **55–72% warns** — heavy overlap worth an
   editorial look; below that, legitimate expansion prose.

## 6. Calibration and results — the worked measurement

The thresholds weren't guessed; they were calibrated against this bundle's own
before/after corpus on 2026-07-13, and the separation is the evidence they sit
in the right place:

- **Before** (the pre-rewrite corpus, 148 term files): the fail band tripped on
  **88 sentences across ~80 files** — more than half the glossary was carrying a
  stacked second definition under its lede.
- **The rewrite:** all 148 bodies were rewritten to expansion-only in the same
  change (links rewoven, never dropped; genuinely-empty bodies left empty rather
  than padded), and the index was materialized — which also fixed pre-existing
  hand-kept ordering drift (e.g. *webhook* filed before *verification grounding*).
- **After:** `mix brain.glossary` passes with **0 failures and 0 warnings** —
  legitimate expansion prose clears the warn band with margin, so a future
  warning is signal, not noise (a standing warning would train the reader to
  ignore the channel — the [spurious-warning](/beliefs/glossary/spurious-warning.md)
  failure mode).
- **Proof grew with it:** 12 new scenario tests (suite 116 → 128), and the check
  now gates CI and the Pages deploy alongside `brain.verify` and
  `brain.route_tags`.

## 7. Semantic limitations — what this check honestly cannot do

- **Paraphrase evades it.** A body sentence that restates the description's
  *meaning* while sharing little of its *vocabulary* scores low and passes. The
  check makes the common failure mode — re-defining the term in nearby words —
  impossible to land silently; it is not a semantic-equivalence oracle, and
  catching a skilled paraphrase stays editorial (review, and the warn band as a
  tripwire).
- **Vocabulary-dense expansion can false-positive into the warn band.** A
  legitimately new point that must reuse the term's core vocabulary (a concrete
  example enumerating the description's own states, say) can score 55–72%. That
  is exactly why the middle band warns instead of fails — the judgment stays
  with the editor, per the same no-mechanical-oracle reasoning as the
  routing-ledger coverage warning.
- **"Repeated aspects" is broader than repeated sentences.** The unit of
  measurement is the sentence; a body that re-covers the description's ground
  diffusely, a clause at a time, can stay under threshold. The authoring rule in
  the skill is the stronger, editorial bar; the check is its mechanical floor.
- **Sub-threshold sentences are unmeasured.** Sentences under 8 content words
  are skipped as too short to score honestly — a terse restatement can hide
  there. Accepted deliberately: gating on 5-word fragments would be noise-bound.

## 8. Verify — the scenario and the gates

**The scenario test pins the engine.**
[`test/second_brain/glossary_test.exs`](/test/second_brain/glossary_test.exs)
builds fixture glossaries in a tmp dir and asserts the contract: a conforming
glossary passes all three checks; a missing description fails; a paraphrased or
reordered index fails with the `--materialize` hint; a stacked reworded
definition fails the repetition check (the red test); expansion-only bodies
pass; citation lines and generated excerpt logs are exempt; moderate overlap
never fails; `materialize/1` regenerates the Terms section while preserving the
prose above it, sorts case-insensitively, and is idempotent once materialized.

**The gate suite** (or just `./.githooks/pre-commit`, which mirrors CI):

```
mix format --check-formatted
mix compile --warnings-as-errors
mix brain.glossary              # descriptions, index sync, body dedup
mix brain.verify
mix brain.route_tags
mix test                        # includes the glossary scenario
```

**The editorial spot-checks the machine can't judge:** is a warn-band sentence
genuine expansion or a vocabulary-shifted restatement; is a description still
the *best* one-paragraph overview of its term; and is an empty body honest
emptiness or a definition that never got its expansion written.
